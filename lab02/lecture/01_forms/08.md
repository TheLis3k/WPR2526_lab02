Formularze `HTML` oferują dwa typy kontrolek do wyboru opcji spośród zestawu możliwości: `<input type="radio">` do wyboru jednokrotnego oraz `<input type="checkbox">` do wyboru wielokrotnego. Oba typy mają podobną strukturę atrybutów, ale różnią się logiką działania.

**Kontrolka `<input type="radio">`** (przycisk opcji) pozwala użytkownikowi wybrać dokładnie jedną wartość z grupy wzajemnie wykluczających się opcji. Grupowanie przycisków radiowych odbywa się przez nadanie im tej samej wartości atrybutu `name`. Jeśli kilka przycisków ma tę samą nazwę, zaznaczenie jednego z nich automatycznie odznacza pozostałe. Atrybut `checked` (bez wartości lub z wartością `checked`) sprawia, że przycisk jest domyślnie zaznaczony:

```html
<input type="radio" name="plec" value="kobieta"> Kobieta
<input type="radio" name="plec" value="mezczyzna" checked> Mężczyzna
```

W powyższym przykładzie obie kontrolki należą do tej samej grupy `plec`. Użytkownik może wybrać tylko jedną z nich. Do serwera zostanie przesłana wartość atrybutu `value` zaznaczonej opcji — np. `plec=mezczyzna`. Jeśli żadna opcja nie jest zaznaczona, pole w ogóle nie pojawi się w tablicy `$_POST` lub `$_GET`.

Przykład z materiałów:

```html
<input type="radio" name="radio1" value="Przycisk nie zaznaczony">
<input type="radio" name="radio2" checked value="Przycisk zaznaczony">
```

**Kontrolka `<input type="checkbox">`** (pole wyboru) pozwala użytkownikowi zaznaczyć zero lub więcej opcji niezależnie od siebie. Każde pole checkbox ma własną nazwę lub — gdy chcemy zebrać wiele wyborów do jednej tablicy — nazwy zakończone nawiasami kwadratowymi (`[]`). Atrybut `checked` powoduje domyślne zaznaczenie pola:

```html
<input type="checkbox" name="zainteresowania[]" value="sport"> Sport
<input type="checkbox" name="zainteresowania[]" value="muzyka" checked> Muzyka
<input type="checkbox" name="zainteresowania[]" value="programowanie"> Programowanie
```

Gdy formularz jest wysyłany, do serwera trafiają wyłącznie zaznaczone pola. Niezaznaczone checkboxy nie pojawiają się w `$_POST` ani `$_GET`. Jeśli użytkownik zaznaczył "Muzyka" i "Programowanie", po stronie `PHP` tablica `$_POST['zainteresowania']` będzie zawierać: `["muzyka", "programowanie"]`.

Przykład z materiałów:

```html
<input type="checkbox" name="check1" value="Przycisk check niezaznaczony">
<input type="checkbox" name="check2" value="Przycisk check zaznaczony" checked>
```

Po stronie `PHP` odczyt wartości pola checkbox (z unikalną nazwą) wygląda następująco:

```php
<?php
if (isset($_POST['check2'])) {
    echo "Pole check2 było zaznaczone, wartość: " . htmlspecialchars($_POST['check2']);
} else {
    echo "Pole check2 nie było zaznaczone.";
}
?>
```

Funkcja `isset()` jest kluczowa przy obsłudze checkboxów: bezpośrednie odczytanie `$_POST['check1']` dla niezaznaczonego pola spowodowałoby błąd `PHP` typu *undefined index*. Użycie `isset()` pozwala bezpiecznie sprawdzić, czy dana wartość w ogóle dotarła do serwera.
