```
2001
0xFE
1.4142
"Hello World"
'Hi'
true
null
```

**Identyfikator** to po prostu nazwa. W `PHP` identyfikatory są używane do nazywania zmiennych, funkcji, stałych i klas. Pierwszym znakiem identyfikatora musi być **litera ASCII** (wielka lub mała), znak podkreślenia (`_`) lub dowolny znak między ASCII `0x7F` a ASCII `0xFF`. Po początkowym znaku obowiązują te znaki i cyfry `0–9`.

**Nazwy zmiennych** zawsze zaczynają się od znaku dolara (`$`) i uwzględniają wielkość liter. Oto kilka poprawnych nazw zmiennych:

```
$bill
$head_count
$MaximumForce
$I_HEART_PHP
$_underscore
$_int
```

Oto kilka niedozwolonych nazw zmiennych:

```
$not valid
$|
$3wa
```

Wszystkie te zmienne różnią się ze względu na rozróżnianie wielkości liter:

```
$hot_stuff $Hot_stuff $hot_Stuff $HOT_STUFF
```

`PHP` udostępnia osiem typów wartości lub typów danych. Cztery z nich to typy skalarne (jednowartościowe): liczby całkowite, liczby zmiennoprzecinkowe, łańcuchy i wartości logiczne. Dwa to typy złożone (kolekcje): tablice i obiekty. Pozostałe dwa to typy specjalne: `resource` i `NULL`.

**Liczby całkowite** to liczby całkowite, takie jak `1`, `12` i `256`. Zakres dopuszczalnych wartości różni się w zależności od szczegółów platformy, ale zwykle rozciąga się od `-2 147 483 648` do `+2 147 483 647`. W szczególności zakres jest równoważny zakresowi typu danych `long` kompilatora `C`. Niestety, standard `C` nie określa, jaki zakres powinien mieć typ `long`, więc w niektórych systemach możemy zobaczyć inny zakres liczb całkowitych. Literały całkowite mogą być zapisywane w postaci dziesiętnej, ósemkowej, binarnej lub szesnastkowej. Wartości dziesiętne są reprezentowane przez sekwencję cyfr, bez zer wiodących. Literał może zaczynać się znakiem plus (`+`) lub minus (`-`). Jeśli nie ma znaku, zakłada się, że jest on dodatni. Przykłady dziesiętnych liczb całkowitych obejmują:

```
1998
−641
+33
```

Liczby ósemkowe składają się z wiodącego `0` i sekwencji cyfr od `0` do `7`. Podobnie jak liczby dziesiętne, liczby ósemkowe mogą być poprzedzone plusem lub minusem. Oto kilka przykładowych wartości ósemkowych i ich odpowiedników dziesiętnych:

```
0755 // decimal 493
+010 // decimal 8
```

Wartości szesnastkowe zaczynają się od `0x`, po którym następuje ciąg cyfr (`0–9`) lub liter (`A–F`). Litery mogą być pisane wielkimi lub małymi literami, ale zwykle są pisane wielkimi literami. Podobnie jak w przypadku wartości dziesiętnych i ósemkowych, możemy dołączyć znak w liczbach szesnastkowych:

```
0xFF // decimal 255
0x10 // decimal 16
–0xDAD1 // decimal −56017
```

Liczby binarne zaczynają się od `0b`, po którym następuje ciąg cyfr (`0` i `1`). Podobnie jak w przypadku innych wartości, możemy dołączyć znak w liczbach binarnych:

```
0b01100000 // decimal 96
0b00000010 // decimal 2
-0b10 // decimal -2
```

Jeśli spróbujemy zapisać zmienną, która jest zbyt duża, aby była przechowywana jako liczba całkowita lub nie jest liczbą całkowitą, zostanie ona automatycznie zamieniona na liczbę zmiennoprzecinkową. Użycie funkcji `is_int()` (lub jej aliasu `is_integer()`), sprawdza, czy wartość jest liczbą całkowitą:

```
if (is_int($x)) {
  // $x is an integer
}
```

**Liczby zmiennoprzecinkowe** (często określane jako liczby „rzeczywiste”) reprezentują wartości liczbowe z cyframi dziesiętnymi. Podobnie jak liczby całkowite, ich ograniczenia zależą od szczegółów naszej maszyny. Liczby zmiennoprzecinkowe `PHP` są równoważne zakresowi typu danych double naszego kompilatora `C`. Zwykle pozwala to na liczby od `1,7E-308` do `1,7E+308` z 15-cyfrową dokładnością. Jeśli potrzebujemy większej dokładności lub szerszego zakresu wartości całkowitych, możemy użyć rozszerzeń `BC` lub `GMP`. `PHP` rozpoznaje liczby zmiennoprzecinkowe zapisane w dwóch różnych formatach (normalnym lub naukowym):

```
3.14
0.017
-7.1
0.314E1 // 0.314*10^1, or 3.14
17.0E-3 // 17.0*10^(-3), or 0.017
```

Wartości zmiennoprzecinkowe są tylko przybliżonymi reprezentacjami liczb. Na przykład w wielu systemach `3.5` jest faktycznie reprezentowana jako `3.4999999999`. Oznacza to, że musimy uważać, aby nie pisać kodu, który zakłada, że liczby zmiennoprzecinkowe są reprezentowane całkowicie dokładnie, na przykład bezpośrednio porównując dwie wartości zmiennoprzecinkowe za pomocą operatora `==`. Normalnym podejściem jest porównanie z kilkoma miejscami po przecinku:

```
if (intval($a * 1000) == intval($b * 1000)) {
  // numbers equal to three decimal places
}
```

Użycie funkcji `is_float()` (lub jej aliasu `is_real()`), sprawdza, czy wartość jest liczbą zmiennoprzecinkową:

```
if (is_float($x)) {
  // $x is a floating-point number
}
```

`PHP` zapewnia operatory i funkcje do porównywania, demontowania, składania, wyszukiwania, zastępowania i przycinania ciągów, a także szereg wyspecjalizowanych funkcji ciągów do pracy z kodowaniami `HTTP`, `HTML` i `SQL`.

**Wartość logiczna** reprezentuje wartość logiczną — mówi, czy coś jest prawdziwe, czy nie. Jak większość języków programowania, `PHP` definiuje niektóre wartości jako prawdziwe (`true`), a inne jako fałszywe (`false`). Głównie ten rodzaj zmiennych występuje jako wynik kodu warunkowego, takiego jak:

```
if ($alive) { ... }
```

W `PHP` wszystkie poniższe wartości mają wartość `false`:

* Słowo kluczowe `false`
* Liczba całkowita `0`
* Wartość zmiennoprzecinkowa `0,0`
* Pusty ciąg (`""`) i ciąg `"0"`
* Tablica z zerowymi elementami
* Wartość `NULL`

Wartość, która nie jest fałszem, to prawda (`true`), w tym wszystkie wartości typu `resource`.

```
$x = 5; // this is a true value
$x = true;
$y = ""; // this is a false value
$y = false;
```

Użycie funkcji `is_bool()`, sprawdza, czy wartość jest logiczna:

```
if (is_bool($x)) {
  // $x is a Boolean
}
```

Wiele modułów zapewnia kilka funkcji do radzenia sobie ze światem zewnętrznym. Na przykład każde rozszerzenie bazy danych ma co najmniej funkcję połączenia z bazą danych, funkcję zapytania do bazy danych i funkcję zamknięcia połączenia z bazą danych. Ponieważ możemy mieć jednocześnie otwartych wiele połączeń z bazą danych, funkcja `connect` udostępnia coś, za pomocą którego można zidentyfikować to unikalne połączenie podczas wywoływania funkcji zapytania i zamykania: `resource` (lub `handle`). Każdy aktywny `resource` ma unikalny identyfikator. Każdy identyfikator jest indeksem liczbowym do wewnętrznej tablicy przeglądowej PHP, która przechowuje informacje o wszystkich aktywnych zasobach. `PHP` przechowuje informacje o każdym zasobie w tej tabeli, w tym liczbę odwołań do zasobu (lub użycia) w całym kodzie. Gdy ostatnie odwołanie do wartości zasobu zniknie, rozszerzenie, które utworzyło zasób, jest wywoływane w celu wykonania zadań, takich jak zwolnienie pamięci lub zamknięcie dowolnego połączenia dla tego zasobu:

```
$res = database_connect(); // fictitious database connect function
database_query($res);
$res = "boo";
// database connection automatically closed because $res is redefined
```

Korzyści z tego automatycznego czyszczenia najlepiej widać w funkcjach, gdy `resource` jest przypisany do zmiennej lokalnej. Gdy funkcja się kończy, wartość zmiennej jest odzyskiwana przez `PHP`:

```
function search() {
  $res = database_connect();
  database_query($res);
}
```

Gdy nie ma więcej odniesień do `resource`, jest on automatycznie zamykany. To powiedziawszy, większość rozszerzeń zapewnia określoną funkcję zamykania lub zamykania i uważa się, że dobrym stylem jest jawne wywoływanie tej funkcji w razie potrzeby, zamiast polegania na zmianie typu zmiennej w celu wywołania czyszczenia.

Użycie funkcji `is_resource()`, sprawdza, czy wartość jest `resource`:

```
if (is_resource($x)) {
  // $x is a resource
}
```

**Callbacks** (tłumaczone jako `wywołania zwrotne`) to funkcje lub metody obiektowe używane przez niektóre funkcje, takie jak `call_user_func()`. Wywołania zwrotne mogą być również tworzone za pomocą metody `create_function()` i przez tzw. closures:

```
$callback = function()
{
  echo "callback achieved";
};
call_user_func($callback);
```

Jest tylko jedna wartość typu danych `NULL`. Ta wartość jest dostępna poprzez słowo kluczowe `NULL` bez rozróżniania wielkości liter. Wartość `NULL` reprezentuje zmienną, która nie ma wartości (podobnie jak `undef` w Perlu lub `None` w Pythonie):

```
$aleph = "beta";
$aleph = null;
$aleph = NULL;
$aleph = Null;
```

Użyj funkcji `is_null()`, aby sprawdzić, czy wartość ma wartość `NULL` — na przykład, aby sprawdzić, czy zmienna ma wartość:

```
if (is_null($x)) {
  // $x is NULL
}
```
