Kontrolka `<input type="text">` to najczęściej stosowany element formularza. Renderuje się jako jednoliniowe pole tekstowe, w którym użytkownik może wpisać dowolny ciąg znaków. Jest używana wszędzie tam, gdzie oczekuje się krótkiego tekstu: imię, nazwisko, adres e-mail, numer telefonu, hasło wyszukiwania itp.

Podstawowa składnia:

```html
<input type="text" name="nazwaPola">
```

Atrybut **`name`** jest kluczowy — to on decyduje o nazwie klucza, pod którym wartość pola będzie dostępna w tablicy `$_GET` lub `$_POST` po stronie `PHP`. Pole tekstowe bez atrybutu `name` nie jest przesyłane do serwera i nie pojawi się w żadnej tablicy superglobalnej.

Atrybut **`value`** pozwala ustawić domyślną wartość pola — tekst, który pojawi się w polu przed jego wypełnieniem przez użytkownika. Jeśli użytkownik nie zmieni zawartości pola, to właśnie ta wartość zostanie przesłana do serwera:

```html
<input type="text" name="miasto" value="Kraków">
```

Atrybut **`size`** określa wizualną szerokość pola wyrażoną w liczbie znaków. Jest to wyłącznie wskazówka dla przeglądarki dotycząca wyświetlania — nie ogranicza ilości tekstu, który użytkownik może wpisać:

```html
<input type="text" name="kod_pocztowy" size="6">
```

Atrybut **`maxlength`** ogranicza maksymalną liczbę znaków, jaką użytkownik może wpisać w pole. Gdy użytkownik osiągnie limit, przeglądarka automatycznie blokuje dalsze wpisywanie:

```html
<input type="text" name="login" value="Wpisz login" maxlength="20">
```

Pełny przykład pola tekstowego z kilkoma atrybutami:

```html
<input type="text" name="input" value="Wypełnij mnie max 15 znaków" maxlength="15">
```

Należy pamiętać, że walidacja po stronie klienta (np. atrybut `maxlength`) jest wyłącznie udogodnieniem dla użytkownika — nie zastępuje walidacji po stronie serwera. Złośliwy użytkownik może ominąć ograniczenia narzucone przez `HTML`, wysyłając żądanie `HTTP` bezpośrednio, z pominięciem przeglądarki. Dlatego w skrypcie `PHP` zawsze należy sprawdzić, czy odebrane dane spełniają oczekiwane kryteria, np. za pomocą funkcji `strlen()` (długość ciągu), `is_numeric()` (czy wartość jest liczbą) lub `filter_var()` z odpowiednim filtrem.

```php
<?php
if (isset($_POST['input'])) {
    $wartosc = $_POST['input'];
    if (strlen($wartosc) > 15) {
        echo "Zbyt długa wartość.";
    } else {
        echo "Odebrano: " . htmlspecialchars($wartosc);
    }
}
?>
```

Wywołanie `htmlspecialchars()` przed wyświetleniem danych jest dobrą praktyką — zamienia znaki specjalne `HTML` (takie jak `<`, `>`, `&`) na ich encje (`&lt;`, `&gt;`, `&amp;`), co chroni przed atakami typu `XSS` (ang. *Cross-Site Scripting*).
