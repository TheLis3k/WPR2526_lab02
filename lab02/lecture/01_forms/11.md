Gdy użytkownik wypełni formularz i kliknie przycisk submit, przeglądarka wysyła dane do serwera. Po stronie `PHP` dane te są dostępne przez tablice superglobalne: `$_GET` — dla formularzy wysłanych metodą `GET`, oraz `$_POST` — dla formularzy wysłanych metodą `POST`. Obie tablice są **tablicami asocjacyjnymi**, gdzie kluczem jest wartość atrybutu `name` kontrolki, a wartością — tekst wpisany lub opcja wybrana przez użytkownika.

Rozważmy przykład formularza z atrybutem `method="POST"`:

```html
<form name="formularz" action="wynik.php" method="POST">
    <input type="text" name="imie" value="">
    <input type="text" name="email" value="">
    <input type="submit" name="wyslij" value="Wyślij">
</form>
```

Plik `wynik.php`, do którego kieruje atrybut `action`, może odczytać dane w następujący sposób:

```php
<?php
if (isset($_POST['wyslij'])) {
    $imie  = $_POST['imie'];
    $email = $_POST['email'];
    echo "<p>Imię: " . htmlspecialchars($imie) . "</p>";
    echo "<p>E-mail: " . htmlspecialchars($email) . "</p>";
}
?>
```

Jeśli formularz korzysta z metody `GET`, jedyna zmiana polega na zastąpieniu `$_POST` przez `$_GET`:

```php
<?php
if (isset($_GET['wyslij'])) {
    $imie = $_GET['imie'];
    echo "Imię: " . htmlspecialchars($imie);
}
?>
```

Dostęp do pola, które nie istnieje w tablicy `$_POST` lub `$_GET`, generuje w `PHP` ostrzeżenie (*undefined index* lub *undefined array key* w nowszych wersjach). Dlatego przed odczytem wartości zawsze zaleca się sprawdzenie jej istnienia funkcją `isset()`:

```php
<?php
$imie = isset($_POST['imie']) ? $_POST['imie'] : '';
?>
```

Skrótowy zapis z operatorem `??` (dostępny od `PHP 7.0`) upraszcza powyższy kod:

```php
<?php
$imie = $_POST['imie'] ?? '';
?>
```

Dla danych przesłanych przez checkboxy i przyciski radiowe zasada jest ta sama. Kluczem jest wartość atrybutu `name`, wartością — zawartość atrybutu `value` zaznaczonej kontrolki:

```php
<?php
// Odczyt przycisku radiowego
$plec = $_POST['plec'] ?? 'brak wyboru';

// Odczyt checkboxa (zaznaczony = klucz istnieje w $_POST)
if (isset($_POST['zgoda'])) {
    echo "Użytkownik wyraził zgodę.";
}
?>
```

Ważną zasadą jest, że dane z formularza nigdy nie powinny być wyświetlane ani przechowywane bez uprzedniego oczyszczenia. Funkcja `htmlspecialchars()` zamienia znaki specjalne `HTML` na ich bezpieczne encje i powinna być używana zawsze przed wyświetleniem jakiejkolwiek wartości pochodnej od danych użytkownika.
